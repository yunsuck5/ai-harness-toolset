- native process invocation that captures stdout / stderr for assertion or downstream processing must go through `Invoke-NativeProcess` (`scripts/lib/native-process.ps1`) — the containment shim that pins `$ErrorActionPreference = 'Continue'` inside its own try / finally, redirects child stdout and stderr to separate temp files, captures `$LASTEXITCODE` immediately after the native invocation, and returns `[pscustomobject]@{ ExitCode; Stdout; Stderr }`. raw `& <native> ... 2>&1` capture-for-use is forbidden in `tests/**/*.ps1` by the `scripts/verify-ps1.ps1` Step F lint; only `$null = & ... 2>&1` (explicit discard), `... 2>&1 | Out-Null` (pipe-to-null discard), and a `# verify-ps1-allow: <non-empty reason>` pragma on the same line bypass the lint. Rationale: under Windows PowerShell 5.1 file-level `$ErrorActionPreference = 'Stop'`, the `& <exe> ... 2>&1` shape wraps each child stderr line as a `NativeCommandError` `ErrorRecord` that fires as a terminating error before `$LASTEXITCODE` can be inspected — the helper removes that race by self-managing EAP, and the lint blocks accidental regression in tests. The underlying PS 5.1 native-command hazard policy remains `docs/policies/POWERSHELL_POLICY.md` "Native command invocation under `$ErrorActionPreference = 'Stop'`"; this entry adds the canonical mitigation primitive (helper + lint) on top.
