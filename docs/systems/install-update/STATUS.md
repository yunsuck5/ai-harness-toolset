- **The install/update/uninstall lifecycle subsystem is in LTS maintenance** (closed at `main` HEAD `69ef433`, confirmatory Pester 440/440; ledger IU-14). The main PC (primary work machine) natural-language lifecycle retest ‚Äî update / uninstall / clean install ‚Äî was subsequently cleared under explicit per-action approval (ledger IU-15).
- **Lifecycle entrypoint split (intended architecture, not a merged mega-CLI):** `scripts/install-global.ps1` (fresh install), `scripts/update-global.ps1` (existing-install update ‚Üí underlying `scripts/install-update.ps1 -Mode update-source`), `scripts/uninstall-global.ps1` (footprint-zero teardown), and `scripts/activate-global.ps1` (3-surface activation apply). Legacy project-local copy mode (channel 5) is backward-compatibility only.
- **Recovery = reinstall-first** (detection-only, deterministic overwrite from trusted source identity = resolved commit SHA); payload integrity = per-file manifest + completeness marker; managed-block paths = replace (`apply-managed-block.ps1`) / remove (`-Remove`) / first-time insertion (`-Insert`).
- Change discipline: further lifecycle changes require a scoped `/goal` + Codex review gate (no drive-by edits); no expansion into an installer-framework / wizard / doctor / repair / health-check class without separate scoped approval (`INSTALL.md` ¬ß11).
- **Deployed runtime extension activation-surface policy (governance, not a behavior change):** `GLOBAL_INSTALL_UPDATE_MODEL.md` ¬ß8A is the single home for the forced-copy + final-verification rule that any source-managed deployed runtime extension (skills, future hooks, future extension types) must be runtime-mirrored and verified ‚Äî payload copy alone is not "installed", a missing/drifted runtime destination after install/update is an install/update failure, and uninstall must reclaim owned surfaces without orphaning. The current concrete instantiation is unchanged (the 3 activation surfaces above; single review-skill resolver). Generic multi-extension enumeration is future scoped work (skills Batch 2C 2C-0), still gated by the change discipline above.
