## Accepted residual risks (LTS maintenance)

Reopen requires concrete new evidence beyond the current pattern.

- **Operator narrow-scope over-confidence** (recurring; dual nature — operator failure mode + review-system effectiveness evidence) — the operator can author scope-narrowed inputs (within-file enumeration, citation provenance, count drift) before Codex review runs; each documented recurrence was mechanically caught by independent Codex review (`Get-ChildItem` / `rg -n` / grep). Mitigation: Codex independent review (primary) + operator discipline in `snippets/claude-skills/ai-harness-review/SKILL.md` step 4 + `REVIEW_RESULT_CONTRACT.md` §5a.4 retraction protocol. No script-side enforcement (governance D12 not-implementable-yet).
- **Bash chain final-exit-code false-fail** (operator reporting convention) — a `;`-chained line ending in an informational non-zero command can mask a preceding `review-verify` success. Harness-level mitigation is out of toolset scope (not the shell-chain reporter's owner); operator convention: do not chain `review-verify` before an informational command whose non-zero exit would override its success.

(The earlier H1-preamble vs `## Counter-argument` surface tension is **resolved**, not an open risk — single-homed in `docs/contracts/review/REVIEW_RESULT_CONTRACT.md` §3c.5.)
