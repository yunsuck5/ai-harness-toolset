# docs/project/ — Project Identity and Positioning

This folder holds **what this project is, is not, and how it is positioned** — read when scoping whether something belongs to the toolset, or understanding its design posture.

| File | Scope |
|---|---|
| `AI_HARNESS_TOOLSET_SCOPE.md` | project nature, in/out of scope, source-vs-target payload mapping, path concepts |
| `TOOLING_POSITION.md` | positioning of adjacent tools (Superpowers, Serena, Codex CLI, ChatGPT Web, etc.) — references, not dependencies |

## What does not belong here

Execution policies (live rules → the active surface `rules/powershell-and-file-encoding.md` + `INSTALL.md` §1 host prerequisites; `docs/policies/` is rationale / history only, not the operative authority), current status (→ per-domain spec/backlog files; question→read-first routing → `docs/README.md`), and decision records (→ the relevant owner surface / domain spec; historical detail in git history).
